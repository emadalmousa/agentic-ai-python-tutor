run_analysis Code — annotiert
tutor_agent.py
def run_analysis(code: str) -> dict:

    llm = get_llm()
    # ↑ Lädt das Ollama-Modell (konfiguriert in config.py)

    tools = _get_tools()
    # ↑ Baut Liste: [explain, debug, exercise] + optional rag_tool

    agent = create_agent(
        llm,
        tools,
        system_prompt=_SYSTEM_PROMPT   # ← HIER kannst du das Verhalten steuern
    )

    result = agent.invoke({
        "messages": [{
            "role": "user",
            "content": code         # ← Der Code des Studenten
        }]
    })
    # ↑ Startet den Loop — LLM + Tools bis keine tool_calls mehr

    text = result["messages"][-1].content
    # ↑ Letzte Nachricht im Loop = finale Antwort des LLM

    return _parse_agent_output(text)
    # ↑ Text → dict mit 5 Feldern (explanation, error_found, ...)
create_agent Parameter — was kannst du ändern

✅ Kannst du ändern

system_prompt
Wichtigster Parameter. Steuert Sprache, Format, Länge der Antwort. In _SYSTEM_PROMPT definiert.
tools
Liste der Tools. Tools hinzufügen oder entfernen in _get_tools().
debug=True
Aktiviert verbose Logging in der Konsole — nützlich beim Entwickeln.
SystemMessage
in jedem Tool
Jedes Tool hat seinen eigenen Prompt. Kurze/lange Antworten werden hier gesteuert.

❌ Nicht anfassen

checkpointer
Speichert Agent-State für Mehrfach-Konversationen. Komplex, nicht nötig.
state_schema
Internes LangGraph-Schema. Sehr komplex, bricht alles wenn falsch gesetzt.
middleware
Für fortgeschrittene Pipelines. Nicht relevant für diesen Use-Case.
response_format
Strukturierter JSON-Output vom LLM. Würde den aktuellen Text-Parser brechen.
interrupt_before/after
Für Human-in-the-Loop Workflows. Nicht nötig hier.

Was sind tool_calls?

Das LLM ruft keine Tools selbst auf — es sagt nur "ich würde gerne dieses Tool aufrufen". LangChain liest das tool_calls-Feld, führt das Tool wirklich aus, und schickt das Ergebnis zurück ans LLM. Der Loop läuft solange bis tool_calls leer ist.

LLM Antwort — OHNE tool_calls (fertig) 
// Loop stoppt hier — das ist die finale Antwort
{
  "content": "Der Code macht eine
  Schleife von 0 bis 4.",

  "tool_calls": []   ← leer = fertig
}
LLM Antwort — MIT tool_calls (Loop weiter) 
// LangChain führt das Tool aus und sendet zurück
{
  "content": "",
  "tool_calls": [{
    "name": "explain_code_tool",
    "args": {
      "code": "for i in range(5)..."
    }
  }]
}
