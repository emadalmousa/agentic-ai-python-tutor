Wie funktioniert's? — Agent-Gedächtnis

memory_service.py — Eigene Memory-Implementierung

⚡ LLM-Aufruf 1 — Antwort mit Kontext

Wenn Lisa eine Nachricht schickt, holt load_memory() ihren Summary aus der Datenbank und packt ihn als Kontext in den System-Prompt. Dann antwortet run_chat() — mit dem Wissen über vergangene Sessions.

⚡ LLM-Aufruf 2 — Memory aktualisieren

Nach der Antwort ruft update_memory() das LLM ein zweites Mal auf: alter Summary plus neue Nachrichten rein — neuer, kompakter Summary raus. Der wird direkt in der DB gespeichert.

Wichtig: Wir speichern nicht den rohen Chat-Verlauf — nur den Summary. Das skaliert viel besser, und jeder User hat seinen eigenen Eintrag in der DB. Pro Nachricht also zwei LLM-Aufrufe — einer für die Antwort, einer für das Gedächtnis-Update.

Wie funktioniert's? — Code-Review Chain

Code-Review Chain — 3 Schritte · 3 LLM-Aufrufe

Lisa schickt ihren Code rein, und dann läuft eine RunnableSequence durch — drei Schritte, drei separate LLM-Aufrufe.

🔴 Schritt 1 — Syntax

Das LLM sucht nur nach echten Fehlern — SyntaxError, NameError, mit Zeilennummer.

🟡 Schritt 2 — Stil

Anderer Prompt, anderes LLM — schaut nur auf PEP8, Naming, Lesbarkeit.

🔵 Schritt 3 — Best Practices

Architektur, Effizienz, ob der Code pythonisch ist.

Der Output von Schritt 1 geht als Input in Schritt 2 — echte Chain, nicht nur drei separate Aufrufe hintereinander. Im Frontend sieht Lisa dann drei aufklappbare Bereiche — rot für Fehler, gelb für Stil, blau für Best Practices.

Wie funktioniert's? — Lernplan-Generator

Personalisierter Lernplan — Basierend auf echten Schwächen

Lisa fragt nach einem Lernplan — und kriegt keinen generischen Wochenplan, sondern einen der auf ihren echten Skill-Scores basiert. Schwache Themen kommen zuerst, max. drei Skills pro Woche.

Der Endpunkt POST /learning-plan liest die gespeicherten Scores aus der DB und übergibt sie dem LLM — das LLM entscheidet dann, in welcher Reihenfolge und welchem Tempo Lisa die Themen angehen soll.

Der Plan ist kein statisches Dokument — er ändert sich, wenn Lisa neue Themen bearbeitet und ihre Scores sich verbessern. Nächste Woche sieht der Plan automatisch anders aus.
