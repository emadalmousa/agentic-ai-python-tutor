🧠 Was das Modell tut
Hier passiert die echte KI-Intelligenz.

Das Modell empfängt System-Prompt + Code und:
• Versteht Python durch Training auf Milliarden Code-Beispielen
• Erkennt Syntaxfehler durch sprachliches Muster-Matching
• Generiert Text Token für Token (Wort für Wort)
• Antwortet auf Deutsch weil der System-Prompt es vorgibt
• temperature=0: kein Zufall → immer gleiche Antwort Neural Network

⚙ FastAPI + subprocess — Code direkt ausführen
📄 Schritt 1: Tempfile anlegen routers/tutor.py → run_code()
tempfile.NamedTemporaryFile(suffix=".py") → schreibt den Code in eine temporäre Datei.
Warum Datei statt direkt? subprocess.run() braucht eine Datei zum Ausführen.
▶ Schritt 2: Python-Interpreter aufrufen
subprocess.run([sys.executable, tmp_path], capture_output=True, text=True, timeout=10)

• sys.executable: derselbe Python der das Backend ausführt (aus dem venv)
• capture_output=True: stdout und stderr werden aufgefangen statt im Terminal auszugeben
• text=True: Output als String (nicht bytes)
• timeout=10: nach 10 Sekunden abbrechen → schützt vor Endlosschleifen subprocess Timeout-Schutz
🗑 Schritt 3: Aufräumen + zurückgeben
os.unlink(tmp_path) löscht die Tempfile (im finally-Block → läuft immer).
Zurück: RunResponse(stdout, stderr, exit_code)
✅ Erfolgreiche Ausführung
{
  "stdout": "Hallo Welt\n",
  "stderr": "",
  "exit_code": 0
}
❌ Laufzeitfehler
{
  "stdout": "",
  "stderr": "SyntaxError: invalid syntax\n...",
  "exit_code": 1
}
Kein LLM involviert! Der Code läuft direkt im Python-Interpreter — das ist schnell (unter 1 Sekunde) und kostet keine API-Credits. Der LLM-Aufruf kommt erst wenn der Nutzer auf "Code analysieren" klickt.

🔑 Jede Schicht — wer ist wofür verantwortlich?
React / Next.js
UI-Rendering, State (useState/useReducer), HTTP-Requests bauen und absenden. Kein KI-Wissen, keine Geschäftslogik.
Custom Hooks
Kapseln State-Logik: loading, error, data. Trennung von UI (Component) und Logik (Hook) — macht Code wiederverwendbar.
FastAPI + Pydantic
HTTP-Server, URL-Routing, automatische JSON-Validierung. Kein KI-Wissen — nur: "kommt der Request an? Hat er die richtigen Felder?"
tutor_agent.py
Orchestriert Tool-Aufrufe sequenziell. Fängt Verbindungsfehler ab. Einzige Stelle die "weiß" dass es 3 Tools gibt.
LangChain
Framework: formatiert Prompts (SystemMessage/HumanMessage), abstrahiert Provider-API. Selbst kein Modell — nur ein strukturierter HTTP-Client.
gpt-4o-mini (Classifier)
Billiges Modell (~0.15$/1M Token) für einfache Ja/Nein-Klassifikation. Verhindert teure gpt-4o-Aufrufe für Off-Topic-Fragen.
Ollama
Lokaler Modell-Server auf Port 11434. Lädt KI-Modelle herunter, stellt OpenAI-kompatible REST-API bereit. Selbst kein KI-Modell.
llama3.2 / gpt-4o
Das eigentliche Neural Network. Versteht Code durch Training, generiert Text Token für Token. temperature=0 = deterministisch.
subprocess (run)
Führt Python-Code direkt im Interpreter aus. Kein LLM! Schnell (<1s), kostenlos. Timeout=10s schützt vor Endlosschleifen.
config.py
Einzige Stelle wo Provider gewählt wird. Automatischer Fallback: OpenAI funktioniert? → nutze es. Sonst Ollama. Kein manuelles Eingreifen nötig.
MarkdownMessage
LLMs antworten in Markdown. Ohne Rendering sieht der Nutzer **rohe Sternchen**. MarkdownMessage macht daraus echte Formatierung.
Pydantic Schemas
schemas.py definiert Ein- und Ausgabe-Formate. FastAPI validiert automatisch — falscher Typ → 422 Error, ohne dass man selbst prüfen muss.
